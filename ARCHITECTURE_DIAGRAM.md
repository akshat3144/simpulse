## Key Design Patterns Used

### 1. **Separation of Concerns**
- Each module has single responsibility
- Physics separate from events
- ML separate from core logic

### 2. **Dataclass Pattern**
- Configuration classes use `@dataclass`
- Type-safe, immutable configs
- Easy to serialize

### 3. **Vector Space Framework**
- All state as numpy arrays
- Efficient batch operations
- Mathematical operations natural

### 4. **Strategy Pattern**
- Multiple AI implementations
- Switchable strategies
- Rule-based vs ML-based

### 5. **Observer Pattern (implicit)**
- Events generate notifications
- Leaderboard observes state changes
- Loosely coupled components
